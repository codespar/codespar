**MIT-licensed self-hostable runtime for commerce AI agents in Latin America.**

CodeSpar is the open-source implementation of the [`@codespar/api-types`](https://www.npmjs.com/package/@codespar/api-types) session contract. Agents written against [`@codespar/sdk`](https://www.npmjs.com/package/@codespar/sdk) (TypeScript) or [`codespar`](https://pypi.org/project/codespar/) (Python) run unchanged against this self-hosted runtime or against the managed tier at `api.codespar.dev` — `baseUrl` is a configuration change, not a rewrite.

The runtime is channel-agnostic and vertical-agnostic in its core abstractions. The **shipped application is LATAM commerce**: agents that transact on Pix, issue NF-e, ship via Melhor Envio, and notify on WhatsApp Business — without rebuilding the plumbing every team.

## What ships in this repo

- **Session API** (`POST /v1/sessions`, `execute`, `send`, `connections`, `DELETE`) — the OSS implementation of the codespar session contract. Fastify, Postgres, Redis.
- **Projects API** (`/v1/projects` CRUD) — 2-level tenancy (Org → Project) mirroring the enterprise contract. Slug rules, default-project enforcement, atomic promotion.
- **WhatsApp channel adapter** via Evolution API. WhatsApp is the deepest channel because Brazilian commerce concentrates there (~78% of businesses, 6× web-e-commerce conversion).
- **Plugin hooks** — `PolicyHook`, `ObservabilityHook`, `SecretsHook`, `IntegrationHook`. The integration surface for the managed tier and any third party.
- **Storage** — `StorageProvider` with `FileStorage` (single-tenant) and `PgStorage` (PostgreSQL via Drizzle) implementations. Audit log is a per-tenant append-only hash chain.

## What's not in OSS yet

The runtime is functionally complete for self-hosting an agent that transacts on LATAM rails — but the following surfaces live only in [`codespar-enterprise`](https://github.com/codespar/codespar-enterprise) and are on the OSS roadmap:

- **MCP server catalog API** (`/v1/servers`). Enterprise has 109 servers in a Postgres-backed catalog with category × country metadata. In OSS, providers are registered manually via SDK config.
- **Connections vault** (`/v1/connections`, `/v1/auth-configs`). Enterprise has an AES-256-GCM vault with per-tenant scrypt-derived keys. In OSS, credentials live in environment variables.
- **Programmable wallet + policy engine + commerce-specific observability + fiscal-compliance certifications.** All managed-tier capabilities. The wallet/policy engine is design-only as of 2026-04 (engineering not yet started).

## Five-point MIT commitment

The MIT layer stays MIT forever. From [VISION §Premise 1a](https://github.com/codespar/codespar-web/blob/main/docs/visions/VISION-codespar.md):

1. Every MCP server already published is MIT forever.
2. Every future MCP server will be MIT — not BSL, SSPL, AGPL, GPL, or "source-available."
3. `@codespar/sdk` stays MIT.
4. No usage-based restrictions on OSS components — no rate limits, no feature gates, no telemetry that restricts functionality.
5. Copyleft is never used as a competitive weapon.

If codespar ceased to exist tomorrow, agent code built on this runtime does not break.

## Quick start

```bash
git clone https://github.com/codespar/codespar
cd codespar
docker compose up   # Postgres + Redis + runtime
```

Point the SDK at it:

```ts
import { CodeSpar } from "@codespar/sdk";

const cs = new CodeSpar({
  baseUrl: "http://localhost:3000",
  apiKey: process.env.CODESPAR_API_KEY,
});

const session = await cs.create("user_123");
```

The same code, with `baseUrl` removed (or set to `https://api.codespar.dev`), runs against the managed tier — which adds the catalog, vault, and governance surfaces listed above.
