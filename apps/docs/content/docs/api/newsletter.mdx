# Newsletter API

CodeSpar includes newsletter subscriber management with optional Resend integration for automated welcome emails. These endpoints power the newsletter signup on the marketing site.

## Subscribe

Add a new subscriber. This endpoint is idempotent -- subscribing an existing email returns success without creating a duplicate.

```
POST /api/newsletter/subscribe
```

### Request

```bash
curl -X POST http://localhost:3000/api/newsletter/subscribe \
  -H "Content-Type: application/json" \
  -d '{"email": "dev@example.com", "source": "homepage"}'
```

### Request Body

| Field | Type | Required | Description |
|-------|------|----------|-------------|
| `email` | string | Yes | Subscriber email address |
| `source` | string | No | Where the subscription originated (e.g., `homepage`, `docs`, `blog`) |

### Response

```json
{
  "email": "dev@example.com",
  "subscribed": true,
  "message": "Subscribed successfully"
}
```

If the email is already subscribed:

```json
{
  "email": "dev@example.com",
  "subscribed": true,
  "message": "Already subscribed"
}
```

### Resend Welcome Email

When `RESEND_API_KEY` and `RESEND_FROM_EMAIL` are configured, a welcome email is automatically sent to new subscribers. The email is sent asynchronously and does not block the subscribe response. If the email fails to send, the subscription is still recorded and a warning is logged.

## List Subscribers

Retrieve all subscribers. Intended for admin use.

```
GET /api/newsletter/subscribers
```

### Request

```bash
curl http://localhost:3000/api/newsletter/subscribers
```

### Response

```json
[
  {
    "email": "dev@example.com",
    "source": "homepage",
    "subscribedAt": "2026-03-21T10:00:00Z"
  },
  {
    "email": "team@company.com",
    "source": "docs",
    "subscribedAt": "2026-03-20T15:30:00Z"
  }
]
```

## Unsubscribe

Remove a subscriber by email.

```
DELETE /api/newsletter/unsubscribe
```

### Request

```bash
curl -X DELETE http://localhost:3000/api/newsletter/unsubscribe \
  -H "Content-Type: application/json" \
  -d '{"email": "dev@example.com"}'
```

### Request Body

| Field | Type | Required | Description |
|-------|------|----------|-------------|
| `email` | string | Yes | Email address to unsubscribe |

### Response

```json
{
  "email": "dev@example.com",
  "subscribed": false,
  "message": "Unsubscribed successfully"
}
```

## Subscriber Count

Get the total number of subscribers. This is a public endpoint suitable for displaying on the marketing site.

```
GET /api/newsletter/count
```

### Request

```bash
curl http://localhost:3000/api/newsletter/count
```

### Response

```json
{
  "count": 142
}
```

## Environment Variables

| Variable | Required | Default | Description |
|----------|----------|---------|-------------|
| `RESEND_API_KEY` | No | -- | Resend API key for sending welcome emails. If not set, subscriptions are stored but no emails are sent. |
| `RESEND_FROM_EMAIL` | No | `CodeSpar <dispatch@codespar.dev>` | From address for newsletter emails. Must be verified in your Resend account. |

## Error Handling

```json
{
  "error": "Invalid email address",
  "code": "INVALID_EMAIL",
  "status": 400
}
```
